# UI 设计规格

## 1. 设计目标

- 现代、克制、轻量
- 与原页面共存，不抢主视觉
- 强调导航效率与长线程可读性

## 2. 视觉风格

- 基调：轻玻璃感、低存在感、细长轨道
- 颜色：中性雾面底色 + 极低饱和节点，只在当前节点使用更明显的强调色
- 阴影：软阴影，避免“悬浮大胶囊”感
- 圆角：高圆角胶囊，但整体足够窄，融入原页面边缘

## 3. 布局

- 时间轴固定在右侧
- 与原生滚动条错开 16 到 28 像素
- 模块总体长度固定，不会因为消息变多而无限变长
- 顶部与底部保留安全留白，避免遮挡站点原生按钮
- 面板内部为纵向细轨道与离散节点

## 4. 节点设计

- 形态：细竖线轨道 + 小体积悬浮圆点
- 总节点数固定为 12 个
- 节点语义依次为：首对话、10% 到 100% 的内容进度、最后回复尾部
- 节点始终沿中轴均匀分布，模块总长度保持不变
- 当前阅读位置节点放大并发光
- 点击节点平滑滚动至对应消息

## 5. 预览设计

- 不使用外部 API
- 采用本地免费方案：截取消息正文首句或前 90 到 120 个字符
- 如果消息主要由代码构成，则展示“代码片段”标签和前若干字符
- 预览层独立于节点渲染，避免鼠标 hover 时出现频闪
- 预览只作为轻提示，不承担完整摘要职责

## 6. 响应式行为

### Desktop

- 轨道完整展示
- 悬浮显示预览气泡

### Mobile

- 面板更窄
- 预览气泡限制宽度并避免超出视口
- 节点点击区域增大，减少误触

### iPad

- 保持右侧纵向时间轴
- 节点间距介于桌面端与手机端之间

## 7. 可访问性

- 所有节点为可聚焦按钮
- 支持键盘导航与焦点态
- 颜色对比满足基本可读性
- 预览气泡使用语义化文本而非纯装饰

## 8. 空态与异常态

- 未识别到消息时不展示面板
- 节点数量过少时仅展示简化轨道
- 提取摘要失败时展示“点击跳转到此消息”

## 9. 设计结论

- 悬浮预览可做，但不依赖 API
- 首版不做复杂摘要算法，避免成本与维护复杂度上升
- 时间轴应始终服务于“定位”和“跳转”，不要演变成第二套聊天 UI
