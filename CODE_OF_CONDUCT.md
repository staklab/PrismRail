# Code of Conduct

**Language / 语言**

[English](#en) | [中文](#zh)

<a id="en"></a>
## English

### Our Goal

PrismRail aims to be a respectful, constructive, and welcoming open-source project.

We want contributors and users to feel comfortable participating in discussions, filing issues, and submitting pull requests.

### Expected Behavior

- Be respectful and professional
- Focus on ideas, bugs, and solutions rather than personal attacks
- Assume good intent and communicate clearly
- Welcome constructive feedback and different technical perspectives
- Help keep discussions useful, calm, and relevant to the project

### Unacceptable Behavior

- Harassment, insults, or discriminatory language
- Repeated hostile or disruptive behavior
- Publishing private or sensitive information without permission
- Spam, trolling, or intentionally derailing discussions

### Scope

This code of conduct applies to:

- GitHub issues
- pull requests
- discussions related to the project
- other project-maintained communication channels

### Enforcement

Project maintainers may edit, hide, lock, or remove content that violates these expectations, and may restrict participation when necessary to protect the project environment.

If you need to report a serious problem, open an issue or contact the maintainer through the repository profile.

## 中文

### 我们的目标

PrismRail 希望成为一个尊重他人、建设性、对协作者友好的开源项目。

我们希望每一位贡献者和使用者都能安心地参与讨论、提交 Issue 和发起 Pull Request。

### 期望行为

- 保持尊重和专业
- 聚焦想法、问题和解决方案，而不是针对个人
- 默认善意，清晰表达
- 欢迎建设性反馈和不同的技术观点
- 共同维护高质量、可协作的讨论氛围

### 不可接受的行为

- 骚扰、侮辱或歧视性言论
- 持续性的敌意或破坏性行为
- 未经允许公开他人的隐私或敏感信息
- 垃圾信息、钓鱼、故意带偏讨论

### 适用范围

本行为准则适用于：

- GitHub Issues
- Pull Requests
- 与项目相关的讨论
- 其他由项目维护的沟通渠道

### 执行方式

项目维护者有权在必要时编辑、隐藏、锁定或移除违反规范的内容，也可以限制相关用户的参与权限，以保护项目协作环境。

如果你需要反馈较为严重的问题，可以提交 Issue，或通过仓库维护者主页联系维护者。
